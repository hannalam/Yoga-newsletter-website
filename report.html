Introduction

What is the essential story being told by your site and what type of structure did you choose to implement?

This website is a hierarchical model structure newsletter for yoga practitioner, in the home page they can find the philosophy and quotes of yoga. Also, there are some introduction sections for other pages.

This website aims to provide information about how yoga practitioner can do at home ranging from doing some simple yoga practice and provide diet tips. For example, under the Yoga Pose page, they can find different poses demonstration with photos and videos. And under the Healthy Diet page, they can find the video about how to make soups.

This website provides demo video for people who consider to buy the yoga training packages. They can join the social community to share their daily practice photos/ videos and experience by logging in to the member page.

In the Shopping page, visitors can go to buy the yoga equipments even they are not member.

State 3 things that have inspired you when creating your website (e.g. guest speakers, websites, artists, blogs).

1. Real experience! I am a verified yoga teacher since 2018 so I would like to use my own experience for this website. I can use what I already know to make the web more realistic.

Accessibility

State 3 ways in which your site is accessible to those with different abilities and needs (for example those with visual impairment)

1. Apply Alt text. I added Alt text for all the images, so when the image is not working or user who are blind or with low vision still know what the images are about.

2. Contrast Colour. I used simple colour style for the website. The background of the website is mainly in white colour, only in the header and footer sections, I used blue colour to highlight the title areas.

3. Videos can be played in the website. To avoid inconvenience that user need to open the video in the new browser, the videos in this website can be play in the same site. And it is not auto play, only when the user click the play button if they would like to play.

Usability

State 3 ways in which you considered the usability of your site.

1.Hidden Nav bar in small screen. I added collapsed navbar for the navigation bar so when the user open in the phone, it can be cleaner and tidier.

2.Font size changed according to screen size. The text in the normal desktop version reminds the same size and changed to smaller when it will be in the phone screen or small screen in order to make the website more responsive.

3.Main page Superlink. To make user friendly, I added surperlink with the logo icon for Header and Footer in every pages, so it can be a short cut to go back to the main page.

Learning

state 3 things you had to learn or find out to create your site. How did you achieve that?

1. Responsive navigation bar. The navigation bar collapses when it is small screen and only show a button to tell the user to open it. When it is in the large screen, it reminds the original design. This navigation bar applied all the pages in the website. I used basic CSS and bootstrap to complete this function. To build up the layout, it took me a while to study as I need to learn it from extra resource.

2. User Login Pop Up. Initially, I just applied a simple alert function to remind user that this login function is coming soon. But later on, I would like to add a bit more complex function to make the website look more professional. This popup flow box allow user to fill user name and password. Also, some button for user to reset password if they forget. I studied it in w3schools.com, it helped a lots.

3. Carousel and thumbnail gallery. The carousel slideshow at Home page show different yoga quote photos, there is auto play or user can swift photos by clicking the left and right arrow button. The thumbnail gallery at Shop page and Wireframe page allow user to click the thumbnail photos and show case the big size of photos. I studied them in w3schools and coursera extra courses.

To learn more skills for the project, I took two extra courses from Coursera.

1 Responsive Website Basics Code with HTML, CSS, and JavaScript
2 Responsive Web Design

I feel more confident to have more basic skills and extra practices after taking those courses.

Evaluation I

What aspects of your work do you think were particularly successful? Why?

The work I feel successful is the javascript code work well in the Shop page. When we click into the sub-navigation bar, one image will show up along with some texts. The format of the content under the sub-nav bar are the same but the data will be changed accordingly.

In the script, it successfully compiles the source code of the template. And render the data into the template and put into the DOM. So, when the user clicks the ‘Yoga Brick’, it will response the result as ‘Title: Yoga brick’, ‘Price: $15’ and ‘Colour choice’.

It is the most difficult part for me in this coursework, I tried many times to adjust the js code as it wasn’t cooperated with other function in the beginning. But after a lot of reading with the course videos, I managed to make it.

I feel satified on the Yogaposes page, when user click the sub nav bar, it changes different photos and content. In the link created by Static Web Page, it didn't show my photos, but with the code editor works well.

Evaluation II

What aspects of your work could be improved? How might you do things differently another time?

There are some small bugs in my website under the Yoga Pose page when it resizes to small screen. The collapse button can’t hide after click on the list. The videos cannot resize to fit the desktop screen. It is probably some issue in the CSS and bootstrap. As I use both of them, it might have some mistakes.

And in the Shop page, the colour buttons in id="detail-template" need to be clicked two time to change colour, then user can click just one time to change colour. If I have more time, I will try different method to fix all the issues.

I managed to separate one of the js data file and link back to the html file, but I didn’t separate in the shop page because this one is more complicated, so next time I will make the code tidier.

I will also make the search bar work in all the pages if there were more time. I have not made it this time as it is very difficult as I tried and failed.

Next time, I will focus more on the mobile phone version with the layout design and the functions. As more user tend to use their phone to browse website, I would like to think more how to implement the touch screen function, such as when the user holds the photo for 3 seconds, it will pop up and flow.
